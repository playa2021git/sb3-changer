# StretchScript to .sb3 Converter MVP

[日本語](README.md) | English

This is a static web app that converts StretchScript generated by Gemini or another AI tool into a Scratch 3.0 / Stretch3-compatible `.sb3` file. You can use it by opening `index.html` in Chrome.

## Safety First

- Extension blocks are never written into `.sb3` files from guessed `opcode`, `inputs`, or `fields` values.
- Blocks registered in `blocks/*.js` must be confirmed from the extension source `getInfo()` and, preferably, from the `project.json` inside a real `.sb3` created in Stretch3.
- If the input contains even one unverified block, conversion does not succeed.
- When unsupported blocks are present, the `.sb3` download button stays disabled.
- Existing Scratch core blocks and the stable Pen path take priority over adding risky extension support.

## How To Use

1. Open `index.html` in Chrome.
2. Paste StretchScript into the editor.
3. Press the conversion button.
4. Download the `.sb3` only after the app reports success.
5. Open the downloaded file in Scratch 3.0 or Stretch3.

Example:

```js
whenGreenFlagClicked(() => {
  penClear();
  penDown();
  repeat(4, () => {
    move(100);
    turnRight(90);
  });
  penUp();
});
```

Expected structure:

- `penClear()` -> `pen_clear`
- `penDown()` -> `pen_penDown`
- `repeat(4, ...)` -> `control_repeat`
- `move(100)` -> `motion_movesteps`
- `turnRight(90)` -> `motion_turnright`
- `penUp()` -> `pen_penUp`
- `extensions` includes `"pen"`

## Regression Coverage

The test suite in `tests/stretch-script.test.mjs` protects the important conversion paths, including:

- Pen square sample
- List creation sample
- Lottery samples B/C/D
- Arithmetic drill
- Basic Sierpinski sample
- Messages, clones, stop blocks, and control blocks
- Costume, backdrop, size, layer, touching, and current-state blocks
- `goTo` expression handling
- `sayNow` followed by `stopAll` warning behavior
- Safe failure for unknown functions
- `ifBlock` patterns used in rock-paper-scissors logic

Run the tests with:

```sh
node --test tests/stretch-script.test.mjs
```

## Microbit More Status

Microbit More support is intentionally limited to the subset that has confirmed save shapes and practical hardware behavior for classroom use.

Currently supported:

- `whenMicrobitButtonPressed("A", () => { ... })`
- `whenMicrobitButtonPressed("B", () => { ... })`
- `ifMicrobitButtonPressed("A", () => { ... })`
- `ifMicrobitButtonPressed("B", () => { ... })`
- `microbitDisplayText("Hello", 120)`
- `whenMicrobitShaken(() => { ... })`
- `whenMicrobitGesture("SHAKE", () => { ... })`
- `microbitPlayTone(440, 100)`
- `microbitStopTone()`

Unsupported until further fixture and hardware verification:

- `TILT_LEFT` / `TILT_RIGHT`
- Servo blocks
- Pin I/O blocks
- Sensor reporter blocks
- BLE or connection-dependent blocks

See [Microbit More block matrix](docs/microbit-more-block-matrix.md), [fixture guide](docs/microbit-more-fixture-guide.md), and [implementation plan](docs/microbit-more-implementation-plan.md) for the verification workflow.

## Notes For AI-Generated StretchScript

When asking an AI tool to generate code for this converter, instruct it to output only StretchScript and to avoid JavaScript APIs such as `parseInt`, `Number`, `toString`, `includes`, `push`, and arrays. The converter treats those as unsupported because Scratch projects should be represented through verified Scratch blocks, variables, and lists.

For a compact prompt-oriented reference, use [stretchscript-spec.md](stretchscript-spec.md).
